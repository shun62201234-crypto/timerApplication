# タイマーアプリ 要件定義（要求仕様）
## 1. 概要

本アプリケーションは、ブラウザ上で動作するタイマーアプリである。
設定した時間をカウントダウンするタイマー機能と、指定した時刻に通知を行うアラーム機能を実行できる。
**ユーザーはタイマー機能とアラーム機能を切り替えて** 利用できる。

## 2. 対象環境

### 2.1  開発環境
-   使用言語: TypeScript
-   使用ツール: Vite, VSCode

### 2.2 動作環境
-   Google Chrome 最新版, Microsoft Edge 最新版

## 3. 機能要求

### 3.1 タイマー
-   時間を設定してカウントダウンできること
-   開始・一時停止・再開・キャンセルができること
-   終了時に通知すること
-   設定した時間を繰り返し利用できること。

### 3.2 アラーム
-   指定時刻に通知できること
-   登録・削除できること
-   ON/OFF切替できること
-   最大5件登録できること

## 4. 非機能要求

### 4.1 データ保存
-   アラーム設定をLocalStorageへ保存すること。

### 4.2 時刻
-   **日本標準時刻**を基準として時刻を判定すること。

## 5. 制約事項
| 項目 | 内容 |
| ---- | ---- |
| 入力方法 | ボタン操作およびスクロール操作のみに限定する |
| 重複登録 | 同一時刻のアラームを複数登録可能とする  |
| 表示言語 | 日本語のみとする |
| Undo機能 | 直前に行った操作や処理を取り消し、元の状態に戻すアンドゥー機能は考慮しない |


# タイマーアプリ 基本設計（機能仕様）
## 1. 概要

本書ではタイマーアプリの機能仕様を定義する。
要件定義で定めた機能要求に基づき、各機能の動作仕様を定義する。

## 2. 機能仕様

### 2.1 共通機能
-   タイマー終了時またはアラーム時刻到達時はアラートを通知する。
-   アラート音は10秒間再生する。
-   アラート終了操作は、アラート表示中のみ受け付ける。
-   アラート終了操作により、アラート音を停止し通知を終了する。
-   音声がミュート状態または音量が0の場合でも、音声再生処理は実行する。
-   ブラウザ起動中のみタイマーおよびアラーム機能を動作させる。
-   アラーム時刻経過後にブラウザを起動した場合、アラーム通知は行わない。
-   保存済みのアラーム設定は保持する。

### 2.2 タイマー機能

#### 2.2.1 時間設定
-   タイマー設定時間の**初期値は「00:00:00」の未設定状態**とする。
-   入力可能な範囲は、**時間(0～23)、分(0～59)、秒(0～59)**とする。
-   設定した時間をタイマー設定時間として設定する。
-   未設定状態と設定済み状態を視覚的に区別する。
-   タイマー設定値が「00:00:00」の場合、**開始操作を受け付けない**。

#### 2.2.2 タイマー開始
-   開始操作により、設定した時間からカウントダウンを開始する。
-   カウントダウン中は開始操作を受け付けない。
-   タイマー終了後は設定時間を保持し、**同じ設定時間で再度開始できる**ものとする。

#### 2.2.3 カウントダウン
-   残時間を1秒ごとに更新する。
-   残時間が「00:00:00」となった場合は、カウントダウンを終了する。

#### 2.2.4 一時停止・再開
-   カウントダウン中は、一時停止操作を受け付ける。
-   一時停止操作を行った場合、残時間を保持したままカウントダウンを停止する。
-   一時停止中は、再開操作を受け付ける。
-   再開操作を行った場合、停止時点の残時間からカウントダウンを再開する。
-   カウントダウン中および一時停止中は、キャンセル操作を受け付ける。

#### 2.2.5 キャンセル
-   カウントダウン中または一時停止中にキャンセル操作を行った場合、タイマーを停止する。
-   タイマーを初期状態「00:00:00」に戻す。

#### 2.2.6 タイマー終了
-   タイマーの残時間が「00:00:00」となった場合、アラートを通知する。

### 2.3 アラーム機能

#### 2.3.1 アラーム登録
-   アラームを登録できる。
-   アラームは**最大5件まで登録**できる。
-   登録件数が5件に達している場合は、新たなアラームを登録できない。

#### 2.3.2 時刻設定
-   アラーム設定時刻の**初期値は「00:00」の未設定状態**とする。
-   入力可能な範囲は、**時間（0～23）、分（0～59）**とする。
-   設定した時刻をアラーム設定時刻として保持する。
-   **「00:00」は有効な時刻**として設定可能とする。

#### 2.3.3 保存
-   設定したアラームを保存する。
-   **初期値「00:00」の状態でも保存できる**。
-   保存したアラームは通知後も保持する。
-   保存したアラームの有効状態の初期値はONとする。

#### 2.3.4 ON/OFF切替
-   保存したアラームの有効状態を切り替えられる。

#### 2.3.5  設定時間経過時の通知
-   **ON状態のアラーム**は、設定時刻に到達した場合にアラートを通知する。
-   OFF状態のアラームは、設定時刻に到達しても通知しない。
-   アラーム時刻は、**日本標準時刻**基準として判定する。

#### 2.3.6 アラート終了
-   アラート終了後も、アラームの有効状態は変更しない。

#### 2.3.7 アラームの削除
-   保存済みのアラームを削除できる。

## 3. エラー処理
-   音声の再生に失敗した場合は、エラー内容をコンソールへ出力する。

## 4. データ保存
-   アラーム設定はLocalStorageへ保存する。
-   削除時はLocalStorageから削除する。

# タイマーアプリ 基本設計（画面仕様）
## 1. 概要

本書ではタイマーアプリの画面仕様を定義する。画面デザインの詳細については別資料を参照する。

## 2. レイアウト

### 2.1 共通
-   タイマー／アラーム切り替えボタンを配置する。

### 2.2 タイマー画面
-   タイマー時間設定スクロールバーを配置する。
-   タイマー開始ボタンを配置する。
-   タイマー残時間表示エリアを配置する。
-   タイマー一時停止／再開ボタンを配置する。
-   タイマーキャンセルボタンを配置する。

### 2.3 アラーム画面
-   アラーム時刻設定スクロールバーを配置する。
-   アラーム追加ボタンを配置する。
-   アラーム保存ボタンを配置する。
-   アラーム設定履歴表示エリア（最大5件）を配置する。
-   アラーム有効／無効切り替えボタン（ON／OFF）を配置する。
-   アラーム削除ボタンを配置する。

### 2.4 アラート画面
-   タイマー終了またはアラーム通知のメッセージを表示する。
- 　アラート終了ボタンを配置する。

## 3. デザイン

### 3.1 デザイン
-   タイマーおよびアラームの時刻設定は、スクロールバーによる入力のみ可能とする。
-   タイマーおよびアラームの設定値は、各項目を**2桁で表示**する(例：01:30:45、00:05)。
-   表示エリアには設定時間または残時間を表示する。
-   ボタンの配置およびサイズはデザイン資料に従う。
-   ボタンのラベルは日本語で表示する。
-   ボタンの表示文は本書で定義した名称を使用する。
-   **アラーム有効／無効切り替えボタンは「ON」「OFF」で表示**とする。
-   無効状態のボタンは灰色で表示し、押下できない状態とする。
-   アラート画面には終了ボタンを配置する。

## 4. 画面仕様

### 4.1 タイマー画面

#### 4.1.1 時間設定スクロールバー入力
-   時間・分・秒をスクロールバーで入力する。
-   設定した時間をタイマー表示エリアへ反映する。

#### 4.1.2 タイマー開始ボタン
-   開始ボタンを配置する。
-   開始ボタンは無効状態を表示できるものとする。
-   無効状態では、開始ボタンを灰色で表示する。

#### 4.1.3 タイマー残時間表示
-   タイマーの残時間を表示エリアへ表示する。

#### 4.1.4 タイマー一時停止/再開切り替えボタン
-   一時停止／再開ボタンを配置する。
-   カウントダウン中は「一時停止」を表示する。
-   一時停止中は「再開」を表示する。
-   利用できない状態では無効表示とする。

#### 4.1.5 タイマー途中キャンセルボタン
-   キャンセルボタンを配置する。
-   カウントダウン中および一時停止中に表示する。

#### 4.1.7 タイマーアラート終了ボタン
-   アラート終了ボタンを配置する。
-   アラート画面に表示する。
-   アラート表示中のみ操作可能とする。

### 4.2 アラーム画面

#### 4.2.1 アラーム追加ボタン
-   アラーム追加ボタンを配置する。
-   登録件数が5件の場合、追加ボタンを無効表示とする。

#### 4.2.2 アラーム数字スクロールバー入力
-   アラーム時刻設定用スクロールバーを配置する。
-   初期表示は「00:00」とする。
-   時間（0〜23）、分（0〜59）をスクロールバーで選択できる。
-   設定した時刻を表示エリアに反映する。

#### 4.2.3 アラーム保存ボタン
-   アラーム保存ボタンを配置する。

#### 4.2.4 アラーム設定履歴表示画面
-   保存済みアラームの一覧を表示するエリアを配置する。
-   アラームは最大5件まで表示する。

#### 4.2.5 アラーム有効／無効切り替えボタン
-   アラームの有効／無効を切り替えるボタンを配置する。
-   ボタン表示はON／OFFで切り替わる。
-   有効状態はON、無効状態はOFFとして表示する。

#### 4.2.7 アラームアラート終了ボタン
-   アラート終了ボタンを配置する。
-   アラート画面に表示する。

#### 4.2.8 アラーム設定済項目の削除ボタン
-   アラーム削除ボタンを配置する。
-   アラーム設定履歴の各項目に表示する。

### 4.3 アラート画面
-   アラートメッセージを表示する。
-   アラート終了ボタンを配置する。

| 保存ボタンを押す     | 履歴一覧に12:30を表示 | ON状態で保存                    |
| 設定時刻到達　 　    | アラート画面　         | 日本標準時12:30にアラート音を再生 |
| 終了ボタンを押す　   | アラーム一覧画面       | ON/OFF状態は変更しない          |
